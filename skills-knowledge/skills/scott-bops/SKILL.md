# Background Operations

A personal operating system that moves repetitive decisions and tasks from conscious
effort into automatic background processes. The core belief: systems beat willpower,
every time.

> "Every area of business and life, from creative writing to routine maintenance,
> can be systematized to reduce cognitive load and eliminate repetitive decision-making."

## When to Intervene

Apply Background Ops frameworks proactively in these situations:

| Scott is... | Apply this framework | How to intervene |
|-------------|---------------------|------------------|
| Planning a new project or feature | Value Stream Mapping | "Before we build, which steps here actually shape the final product?" |
| Evaluating an opportunity or client | Opportunity Evaluation | "Let's run this through the Admiration Test and Hell Yeah or No." |
| Designing a process or automation | EIB Protocol | "Let's walk through Explicate → Improve → Backgroundize." |
| Stuck on writing, content, or creative work | Pozen's Creative Process | "Let's stop creating and drop back to outlining. What are you trying to say?" |
| Debugging a recurring problem | Universal Principles | "This keeps happening. Let's spend 15 minutes diagnosing the root cause." |
| Discussing habits, routines, or tracking | Keystone Calibration | "What's your success rate? Let's calibrate using the 70% rule." |
| Feeling overwhelmed, scattered, or burned out | Explore/Exploit + Failure Modes | "Sounds like reactance or overcomplexity. Let's simplify." |
| Making a binary yes/no decision | Hard Rules | "Is there a rule for this, or should we create one?" |
| Running or planning a meeting | Meeting Protocol | "Let's set a strict agenda with 10-minute decision checkpoints." |
| Installing a new behavior or process | Operational Entrainment | "Let's codify this, account for human nature, then drill it." |
| Under stress or in crisis mode | Anti-Panic Protocol | "What do you have to get done? Just that. Focus there." |

**Intervention style:** Brief and natural. Don't dump the whole framework. One sentence
that names the relevant framework and asks the right question. Only go deeper if Scott
engages with it.

## The Meta-System

These frameworks form a sequential system. Each layer depends on the one before it:

```
1. Map the Value Stream     → Identify what creates value vs. waste
   ↓
2. Explicate-Improve-Backgroundize → Streamline and automate the valuable parts
   ↓
3. Ops Keystone (Lights Spreadsheet) → Track that new habits are actually maintained
   ↓
4. Operational Entrainment  → Drill behaviors that can't be automated
   ↓
5. Universal Principles     → Extract permanent rules from edge cases
```

Never skip ahead. Don't automate a broken process. Don't track a habit you haven't
streamlined. Don't drill a behavior you haven't written down.

## The 10 Frameworks (Quick Reference)

Full step-by-step processes and examples are in `references/frameworks.md`.
Coaching prompts for each context are in `references/coaching-prompts.md`.

### 1. Value Stream Mapping
**Purpose:** Audit any workflow by separating value-producing steps from waste.
**When:** Before building any automation or process.
**Key insight:** In unoptimized systems, up to 90-98% of steps are waste.
**Process:** Draw out every step → Mark which steps directly shape the final product → Eliminate everything else.

### 2. EIB Protocol (Explicate → Improve → Backgroundize)
**Purpose:** The core loop for turning manual work into automatic processes.
**When:** Any time a process needs to be systematized.
**Critical rule:** Never automate a broken process. Explicate before improving. Improve before backgroundizing.

### 3. The Lights Spreadsheet (Ops Keystone)
**Purpose:** Self-calibrating daily habit tracker targeting 70% weekly success.
**The 70% Rule:**
- >70% → increase difficulty or add new habits
- <70% for two consecutive weeks → make it easier
- 100% means the system is too easy and you're stagnating
- Must survive your worst week (5-10 min max daily)

### 4. Explore/Exploit Framework
**Purpose:** Know when to capitalize on known gains vs. discover new opportunities.
**Key signals:**
- Boredom/fatigue = biological signal to shift from exploit to explore
- Morning = exploit (fresh energy for decisive action)
- Evening = explore (natural fatigue prompts trying new things)
- Max sustainable pace: 60% of maximum output

### 5. Pozen's Creative Process
**Purpose:** Prevent cognitive overload by strictly separating creative phases.
**Process:** Brainstorm → Categorize → Outline → Create → Edit (never simultaneously)
**When stuck:** Stop creating. Ask "What am I trying to say here?" Drop back to outlining.

### 6. Operational Entrainment
**Purpose:** Install behaviors so deeply they require zero thought, even under stress.
**Process:** Codify → Account for human nature → Strip inessentials → Drill → Inspect under stress
**Key insight:** Boring, repetitive duties get neglected precisely because nothing usually goes wrong. Entrainment prevents this.

### 7. Universal Principles
**Purpose:** Turn recurring problems into permanent rules.
**Process:** Diagnose root cause (15-60 min) → Codify problem → Write universal rule → Operationalize → Entrain
**Critical rule:** Diagnosis must happen BEFORE prescription. Never jump from problem to solution.

### 8. Hard Rules & Soft Guidelines
**Purpose:** Replace ambiguous decisions with binary, non-negotiable rules.
**Philosophy:** Hard rules > soft guidelines because they remove ambiguity entirely. You don't waste energy deciding IF you should follow a hard rule.
**Key rules:**
- Hell Yeah or No (less than 10/10 excitement = no)
- The Admiration Test (would you vacation with this person after 14 intense days?)
- Clean Separation (one device for work, different device for leisure)
- Anathema (completely forbid specific bad habits, no moderation)
- The Worst Week Rule (system must survive your worst week)
- Error Logging (log mistakes = safe; hide mistakes = trouble)

### 9. The Issue/Error Log
**Purpose:** Track errors transparently to improve the system, not punish the person.
**Key insight:** Firing someone for a mistake causes everyone to hide mistakes. Log errors → improve the machine.

### 10. Meeting Protocol
**Purpose:** Prevent meetings from degrading into unstructured time-wasting.
**Process:** Set strict agenda → 10-minute timer → Write down what was decided → Repeat.
**Key metric:** If anyone spends 25%+ of their time in meetings, the organization has a structural problem.

## Daily Cadence

**Morning (Exploit Mode):** Execute tracked habits. Dive into value-producing work.
Use fresh cognitive capacity for decisive action. Cap at 60% of maximum output.

**Midday (Sentry Duty):** Handle routine checks, client responses, admin. Rely on
Hard Rules to avoid decision fatigue. Run meetings with strict protocol.

**Evening (Explore Mode):** Use natural fatigue as a signal to explore (learn, network,
try new things). Update tracking (5-10 min max). Plan tomorrow.

## Failure Modes

Watch for these patterns and intervene when you see them:

1. **Overcomplexity:** System takes more than 10 minutes daily → too complex. Simplify.
2. **Prescribing before diagnosing:** Jumping to solutions without root cause analysis. Slow down.
3. **Quitting the favorite vice first:** Build the quitting skill on easy wins first.
4. **Psychological reactance:** Rebelling against own rules. Under-schedule during first weeks of any new system.
5. **Panic fixation:** Under stress, rational thinking shuts down. Entrained response: "What do I have to get done?"

## Metrics That Matter

| Metric | Target | Cadence | What It Reveals |
|--------|--------|---------|-----------------|
| Keystone success rate | 70% | Weekly | Calibration (too easy = stagnating, too hard = burning out) |
| Error log frequency | Track all | Immediate | Systemic flaws in processes, not bad people |
| Meeting time % | <25% | Monthly | Organizational health |
| Value vs. waste ratio | Minimize waste | Per process audit | True efficiency of any workflow |
