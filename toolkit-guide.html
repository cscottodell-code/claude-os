Why hooks matter more than rules

CLAUDE.md instructions are ~80% reliable. Hooks are 100%. If something must happen every time, it's a hook. If it's guidance Claude should consider, it's a rule.

HookTypeWhenWhat It Does
guard-phase-completion.sh Guard PreToolUse Blocks "phase complete" unless closeout marker exists. The most important hook in the toolkit.
guard-git-push.sh Guard PreToolUse Prevents accidental pushes to protected branches.
guard-destructive.sh Guard PreToolUse Blocks destructive git operations (reset --hard, clean -f, etc.).
guard-npm-install.sh Guard PreToolUse Enforces pnpm. Catches stack drift when packages change.
guard-claude-md.sh Guard PreToolUse Protects CLAUDE.md from accidental overwrites.
session-start.sh Session Session Start Discovers context files, detects project category, triggers AUTO-RESUME if resume file exists.
session-end.sh Session Session End Reminds to update CLAUDE.md, todo.md, lessons.md.
pre-compact.sh Context PreCompact Saves .context-snapshot.md before compaction so nothing is lost.
extract-instincts.sh Context PreCompact / Stop Captures session patterns to instinct-candidates.md for later review.
context-reminders.sh Context PostToolUse Warns at 60min (then every 30min) and 100 tool uses. Catches context rot.
offload-large-output.sh Context PostToolUse Saves tool output >4KB to overflow files instead of filling context window.
auto-format.sh Quality PostToolUse (Write/Edit) Runs Prettier on .js/.ts/.vue/.css/.json files after every edit.
pre-completion-checklist.sh Quality Stop Checks for uncommitted changes, stale todo.md, missing resume file.
uiux-reminder.sh Quality PostToolUse Nudges to run /impeccable:audit when .vue files change during GSD execution.
check-file-test-trigger.sh Quality PostToolUse Auto-runs test fixtures when check files in checks/ are edited.
post-commit-skill-triggers.sh Quality Post-Commit Suggests relevant skills after git commits.

The toolkit gets smarter every time you use it

Lessons from each project feed back into checks that apply to ALL future projects. This is the single most valuable thing in the toolkit, and it needs reps to compound.

1. Build with the toolkit

Use /scott:new-project, /scott:new-feature, or /scott:resume to start work. GSD + Superpowers handle execution.

↓ phase completes

2. Phase closeout (mandatory, hook-enforced)

Run /scott:phase-closeout. Verify tests pass. Stack audit runs checks. Code review catches issues. Reflection interview captures errors, successes, and lessons.

↓ lessons tagged with [stack], [pattern], [prompt], [harness], [project]

3. Lessons accumulate

Error logs go to errors/. Success logs go to successes/. Retros go to retros/. Lessons go to tasks/lessons.md. The retro index summarizes cross-project patterns.

↓ [stack]-tagged lessons become check candidates

4. Stack review dashboard

Run /scott:stack-review. See system health, check candidates (lessons waiting to become checks), health report, refinement suggestions, and overlap detection.

↓ Scott approves promotions

5. Lessons become checks

Approved [stack] lessons get promoted into check files (checks/*.json). These checks now run automatically during every future phase closeout's stack audit.

↓ next project starts smarter

6. Spa day (monthly cleanup)

Run /scott:toolkit-spa-day. Audit state, review instinct candidates, scan for contradictions, consolidate, verify context budget, run stack review. Keeps the toolkit lean.

↓ push to GitHub, every machine benefits
